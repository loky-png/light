# Инструкция по развертыванию исправлений

## 🚀 Быстрый старт

### 1. Остановите текущий сервер
```bash
# Найдите процесс сервера
ps aux | grep node

# Остановите его
kill <PID>
```

### 2. Обновите зависимости (если нужно)
```bash
cd light-server
npm install

cd ../light-client
npm install
```

### 3. Пересоберите клиент
```bash
cd light-client
npm run build
```

### 4. Запустите сервер
```bash
cd light-server
npm run dev
# или для production
npm start
```

### 5. Запустите клиент (для разработки)
```bash
cd light-client
npm run dev
```

## 🔍 Мониторинг

### Проверка логов сервера
```bash
# Смотрите логи в реальном времени
tail -f light-server/logs/server.log

# Или если используете PM2
pm2 logs light-server
```

### Ключевые сообщения в логах:
- `✅ Socket connected` - пользователь подключился
- `📨 New message` - новое сообщение
- `Broadcasting message to chat room` - сообщение отправлено в комнату
- `[ChatWindow] New message received` - сообщение получено клиентом
- `Message for different chat, ignoring` - сообщение отфильтровано (это нормально)

## ⚠️ Возможные проблемы

### Проблема: Сообщения не доставляются
**Решение:**
1. Проверьте что сервер запущен
2. Проверьте что клиент подключен к правильному URL
3. Проверьте логи на наличие ошибок
4. Убедитесь что firewall не блокирует порт 80

### Проблема: Старые сообщения не загружаются
**Решение:**
1. Проверьте что база данных не повреждена
2. Проверьте индексы: `sqlite3 light.db ".indexes"`
3. Пересоздайте индексы если нужно

### Проблема: Высокая нагрузка на сервер
**Решение:**
1. Проверьте количество онлайн пользователей
2. Проверьте размер базы данных
3. Запустите cleanup вручную: см. функцию `cleanupOldMessages()`
4. Рассмотрите использование Redis для онлайн статусов
