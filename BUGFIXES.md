# Исправленные баги и проблемы в Light Messenger

## 🔴 Критические проблемы

### 1. ✅ Сообщения показывались не в тех чатах
**Проблема:** Пользователи получали сообщения из всех чатов, в которых они состоят, а не только из текущего открытого.

**Причина:** 
- В `ChatWindow.tsx` использовался `socket.removeAllListeners()` который удалял ВСЕ обработчики
- Фильтрация по `chatId` происходила на клиенте, но обработчики дублировались

**Решение:**
- Убрал `removeAllListeners()` и заменил на точечную отписку от конкретных обработчиков
- Добавил логирование для отладки
- Socket.IO rooms уже обеспечивают правильную доставку, нужно было только правильно управлять подписками

**Файлы:** `light-client/src/components/ChatWindow.tsx` (строки 28-80)

---

### 2. ✅ Race condition при создании чата
**Проблема:** Если два пользователя одновременно создавали чат друг с другом, могло создаться 2 чата.

**Причина:** Между проверкой существования чата и его созданием проходило время, в течение которого другой запрос мог создать чат.

**Решение:**
- Обернул проверку и создание в транзакцию `db.transaction()`
- Теперь операция атомарная и race condition невозможен

**Файлы:** `light-server/src/index.ts` (строки 147-220)

---

## 🐛 Важные баги

### 3. ✅ Утечка памяти в onlineUsers Map
**Проблема:** При disconnect пользователи не удалялись из Map, только обновлялся lastSeen. Map росла бесконечно.

**Решение:**
- Добавил автоматическое удаление из Map через 5 минут после disconnect
- Используется setTimeout с проверкой что пользователь не переподключился

**Файлы:** `light-server/src/index.ts` (строки 340-355)

---

### 4. ✅ Неправильная проверка прав на удаление сообщений
**Проблема:** Проверялось только членство в чате, но не проверялось что сообщение принадлежит этому чату. Теоретически можно было удалить чужое сообщение из другого чата.

**Решение:**
- Добавил проверку `WHERE id = ? AND chat_id = ?` при получении сообщения
- Теперь невозможно удалить сообщение из другого чата

**Файлы:** `light-server/src/index.ts` (строки 304-345)

---

### 5. ✅ SQL Injection в cleanupOldMessages
**Проблема:** При большом количестве старых сообщений (>1000) динамическое построение placeholders могло привести к проблемам с производительностью или ошибкам.

**Решение:**
- Добавил пакетную обработку по 500 сообщений
- Обернул в транзакцию для атомарности
- Улучшил производительность

**Файлы:** `light-server/src/index.ts` (строки 230-255)

---

### 6. ✅ SQL Injection в удалении чата
**Проблема:** Аналогичная проблема с динамическими placeholders при удалении чата с большим количеством сообщений.

**Решение:**
- Добавил пакетную обработку по 500 сообщений
- Обернул всю операцию в транзакцию

**Файлы:** `light-server/src/index.ts` (строки 120-145)

---

## 🔧 Улучшения производительности

### 7. ✅ Добавлены индексы для оптимизации запросов
**Проблема:** Некоторые частые запросы не имели индексов.

**Решение:**
- Добавил составной индекс `idx_messages_chat_created` для запросов по чату и времени
- Добавил индекс `idx_chat_members_chat_id` для быстрого поиска участников

**Файлы:** `light-server/src/db.ts` (строки 35-42)

---

### 8. ✅ Валидация длины поискового запроса
**Проблема:** Не было ограничения на длину поискового запроса, что могло привести к проблемам с производительностью.

**Решение:**
- Добавил проверку максимальной длины запроса (50 символов)
- Возвращается ошибка 400 при превышении

**Файлы:** `light-server/src/index.ts` (строки 105-118)

## 🧪 Рекомендации по тестированию

### Тест 1: Проверка изоляции сообщений
1. Создайте 2 чата с разными пользователями
2. Откройте оба чата в разных окнах
3. Отправьте сообщение в чат A
4. Убедитесь что сообщение НЕ появилось в чате B

### Тест 2: Проверка race condition
1. Откройте 2 окна с разными пользователями
2. Одновременно создайте чат друг с другом
3. Убедитесь что создался только 1 чат

### Тест 3: Проверка утечки памяти
1. Подключитесь и отключитесь 100 раз
2. Проверьте что Map не растет бесконечно (через логи)

### Тест 4: Проверка прав на удаление
1. Попробуйте удалить сообщение из другого чата (через API)
2. Убедитесь что возвращается ошибка
